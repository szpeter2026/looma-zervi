> **受众：** Jason（前端）、@szbenyx（后端）、内测志愿者协调人  
> **关联：** [INTERNAL_TEST_READINESS.md](./INTERNAL_TEST_READINESS.md) · [MCP底座交付说明_内测.md](./MCP底座交付说明_内测.md)  
> **代码真源：** `backend/src/api/routes/analytics_routes.py` · `frontend/packages/shared-core/src/analytics/track.ts`

---

## 1. 定位与边界

| 项目 | 说明 |
|------|------|
| **目标** | PlanetX → T空间 → HR 闭环的内测可观测；回答「用户卡在哪一步」 |
| **范围** | PlanetX Web、T空间 Web、小程序（经 shared-core） |
| **不在范围** | Bolent 门户（`szbolent-portal`）暂不接；应用排障日志走 structlog，不写入 `product_events` |
| **存储** | SQLite `product_events` / `micro_feedback` 表（Looma `looma.db`） |

**成熟原则：** 产品埋点 ≠ 应用日志 ≠ APM。本方案只做**产品行为事件**与**一步微反馈**。

---

## 2. 架构

```text
PlanetX / T空间 / 小程序
    │  shared-core trackEvent() → 批量 flush（300ms）
    ▼
POST /v1/analytics/events     optional_auth，最多 50 条/批
    ▼
SQLite product_events
    ▼
GET /v1/analytics/funnel      require_auth，管理查看聚合
    │
    └── scripts/funnel-report.sh（命令行报告）
```

**备案前 CVM（IP 内测）同源访问：**

| 能力 | URL 示例 |
|------|----------|
| 埋点上报 | `http://1.14.202.161/v1/analytics/events` |
| 漏斗查询 | `http://1.14.202.161/v1/analytics/funnel?days=30` |
| 微反馈 | `http://1.14.202.161/v1/feedback/micro` |

经 Nginx 反代 `/v1/` → `127.0.0.1:5200`，与 PlanetX `/`、T空间 `/tspace/` 共用同一 IP。

## 3. API 契约

### 3.1 批量上报 `POST /v1/analytics/events`

**鉴权：** 可选（`optional_auth`）；已登录时自动补 `user_id`。

**请求体：**

```json
{
  "events": [
    {
      "event_name": "trial_clicked",
      "session_id": "sess_xxx",
      "platform": "planetx_web",
      "share_code": "ABC123",
      "success": true,
      "properties": { "plan_id": "pro" }
    }
  ]
}
```

**响应：** `{"ok": true, "ingested": 1}`

**约束：**
- `events` 非空数组，单次 ≤ 50 条
- `event_name` 必须在服务端白名单（见 §4）
- 未知事件名 → `400`

### 3.2 漏斗统计 `GET /v1/analytics/funnel?days=30`

**鉴权：** 必须（`Authorization: Bearer <JWT>`）

**参数：** `days` 1–90，默认 30

**响应结构：**

```json
{
  "days": 30,
  "steps": {
    "quiz_complete": 0,
    "share_code_created": 0,
    "share_link_copied": 0,
    "profile_view_public": 0,
    "hr_register_from_share": 0,
    "candidate_imported": 0,
    "trial_started": 0
  },
  "conversion": {
    "share_after_quiz": null,
    "view_after_share": null,
    "import_after_view": null,
    "trial_after_import": null
  },
  "micro_feedback": [],
  "other_events": { "trial_clicked": 1 }
}
```

- `steps`：主漏斗 7 步计数
- `conversion`：逐步转化率（分母为 0 时 `null`）
- `other_events`：白名单内但不在主漏斗的步骤（如 `trial_clicked`、`funnel_drop`）

### 3.3 微反馈 `POST /v1/feedback/micro`

**鉴权：** 可选

```json
{
  "context": "planetx_result",
  "score": 5,
  "optional_text": "结果页很有帮助",
  "session_id": "sess_xxx",
  "platform": "planetx_web"
}
```

- `context`：`planetx_result` | `tspace_profile_share` | `tspace_pricing`
- `score`：0–5（0=👎，1=👍，或 1–5 量表）

## 4. 事件白名单

与 `backend` `VALID_EVENTS` 及 `shared-core/constants/analytics.ts` **保持同步**。

| event_name | 触发场景 | 主漏斗 |
|------------|----------|--------|
| `quiz_complete` | PlanetX 人格测试完成 | ✅ |
| `share_code_created` | 求职者生成画像分享码 | ✅ |
| `share_link_copied` | 复制分享链接 | ✅ |
| `profile_view_public` | HR/访客打开公开画像页 | ✅ |
| `profile_view_failed` | 画像页加载失败 | — |
| `hr_register_from_share` | HR 从分享链路注册 | ✅ |
| `candidate_imported` | HR 导入候选人 | ✅ |
| `candidate_import_duplicate` | 重复导入 | — |
| `trial_started` | Stub/真实开通试用 | ✅ |
| `trial_failed` | 试用开通失败 | — |
| `trial_clicked` | 点击试用按钮 | — |
| `funnel_drop` | 明确放弃（带 `properties.step`） | — |

**平台枚举：** `planetx_web` | `planetx_mp` | `tspace_web` | `unknown`

## 5. 闭环漏斗（内测核心）

```text
quiz_complete
    → share_code_created
        → profile_view_public
            → candidate_imported
                → trial_started
```

**管理侧关注转化率（`conversion` 字段）：**

| 键 | 含义 |
|----|------|
| `share_after_quiz` | 测完人格后创建分享码的比例 |
| `view_after_share` | 分享码创建后被查看的比例 |
| `import_after_view` | 查看后被 HR 导入的比例 |
| `trial_after_import` | 导入后开通试用的比例 |

## 8. 部署验收清单

内测 Ready 埋点相关项：

- [ ] `POST /v1/analytics/events` 返回 `ingested >= 1`
- [ ] 非法 `event_name` 返回 `400`
- [ ] 已登录用户 `GET /v1/analytics/funnel` 返回含 `steps` 的 JSON
- [ ] `POST /v1/feedback/micro` 返回 `201`
- [ ] PlanetX / T空间页面操作后，24h 内 `funnel-report.sh` 对应步骤计数增加

## 9. 变更规则（防报表腐烂）

1. **新增事件** → 同时改：`analytics_routes.py` `VALID_EVENTS`、`shared-core/constants/analytics.ts`、本文档 §4
2. **禁止**业务组件直接 `fetch('/v1/analytics/events')`
3. **主漏斗步骤**变更 → 同步 `db/manager.py` `get_funnel_stats()` 中 `funnel_events` 列表
4. P1 规划：`event_id` UUID 去重、`occurred_at` 客户端时间、漏斗 API 限 `admin` 角色

## 10. 隐私与留存

- 不在 `properties` 存邮箱、手机号明文
- `session_id` 使用随机串（`sess_*`），非设备指纹
- 内测数据留存建议：原始事件 90 天；聚合漏斗可长期
- 隐私政策需写明行为数据收集（见 `TENCENT_CLOUD_COMMERCE.md` P0-17）

## 11. 常见问题

| 现象 | 原因 | 处理 |
|------|------|------|
| `funnel` 返回 401 | 未带 JWT | 先 `POST /v1/auth/login` |
| 上报成功但 `steps` 仍为 0 | 事件在 `other_events` | 查 `trial_clicked` 等非主漏斗事件 |
| PlanetX 无上报 | 未 `initAnalytics` | 检查 `usePlanetXAnalytics` 是否挂载 |
| CORS 报错 | 跨域调 API | IP 内测应同源 `/v1/`；或配置 `CORS_ORIGINS` |
