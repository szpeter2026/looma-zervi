# 内测埋点与闭环漏斗 — 实施方案

> **版本**：v1.2 ｜ **日期**：2026-07-01  
> **目标**：支撑内测两大诉求 — ① 验证最小闭环路径 ② 收集 UX 行为与轻量反馈  
> **关联文档**：[最小闭环执行计划.md](./最小闭环执行计划.md) · [产品介绍与内测说明.md](./产品介绍与内测说明.md)（第二节：身份与商业模式）

---

## 一、背景与问题

| 诉求 | 现状缺口 |
|------|---------|
| 验证最小闭环 | 业务 API 可通（`verify-closed-loop.sh`），但**无漏斗转化率**数据 |
| UX 与行为数据 | 叙事 `/v1/narrative/*` 较完整；**闭环各步无统一事件**；Chat 评分 API 未接 UI |

**原则**：备案期内自研、`product_events` 落 SQLite，服务端关键节点**必记**（防前端漏报），前端补记交互细节。

---

## 二、标准事件字典（闭环漏斗）

| event_name | 触发位置 | 来源 | 优先级 |
|------------|---------|------|--------|
| `quiz_complete` | profile-sync 成功且含 personality_type | 服务端 | P0 |
| `share_code_created` | referral create purpose=profile_share | 服务端 | P0 |
| `share_link_copied` | SharePanel / 小程序复制 T-space 画像链接 | 前端 | P0 |
| `profile_view_public` | GET profile-view 成功 | 服务端 | P0 |
| `profile_view_failed` | profile-view 404/错误 | 服务端 | P0 |
| `hr_register_from_share` | register 且 query 含 `from=share`（**历史事件名**；语义：经画像链路的个人注册，非 HR 专用账号） | 前端 | P0 |
| `candidate_imported` | import-share 成功 imported=true | 服务端 | P0 |
| `candidate_import_duplicate` | import-share 已存在 | 服务端 | P1 |
| `trial_started` | payment/upgrade → pro 成功 | 服务端 | P0 |
| `trial_failed` | payment/upgrade 失败 | 服务端 | P1 |
| `micro_feedback` | 微反馈提交 | 服务端（micro_feedback 表） | P0 |

**公共字段**：`session_id`, `user_id`, `platform`, `share_code`, `properties` (JSON), `source` (`client`|`server`)

建议 `properties` 中区分上下文（若前端可传）：`context`: `personal` | `enterprise`；企业侧事件可带 `enterprise_id`（见 [产品介绍与内测说明.md](./产品介绍与内测说明.md) §2.4）。

**platform 枚举**：`planetx_web` · `planetx_mp` · `tspace_web` · `unknown`

## 三、技术方案

### 3.1 后端

| 项 | 说明 |
|----|------|
| 表 `product_events` | 通用行为事件 |
| 表 `micro_feedback` | 上下文微反馈（👍/👎、1–5 分） |
| `POST /v1/analytics/events` | 批量上报（optional_auth，支持匿名 session） |
| `POST /v1/feedback/micro` | 微反馈 |
| `GET /v1/analytics/funnel` | 内测漏斗汇总（require_auth） |
| `src/analytics/events.py` | `log_product_event()` 供各 route 调用 |
| 服务端 hook | referral create/view、enterprise import、payment upgrade、game profile-sync |

### 3.2 shared-core

| 项 | 说明 |
|----|------|
| `types/analytics.ts` | 事件与反馈类型 |
| `analytics/track.ts` | `getAnalyticsSessionId()`、`trackEvent()` |
| `createAnalyticsApi()` | events 批量、micro 反馈、funnel 查询 |
| `CLOSED_LOOP_EVENTS` 常量 | 事件名集中管理 |

### 3.3 前端埋点（P0 触点）

| 端 | 文件 | 动作 |
|----|------|------|
| PlanetX Web | `SharePanel.tsx` | `share_link_copied` |
| PlanetX Web | `ResultScreen.tsx` | 微反馈「像你吗？」 |
| T-space | `CandidateShare.tsx` | 微反馈「信息够用吗？」 |
| T-space | `Register.tsx` | `hr_register_from_share`（见上表语义说明） |
| T-space | `Pricing.tsx` | 前端补记 `trial_clicked`（服务端记 trial_started） |
| 小程序 | `result/index.ts` | `share_link_copied` |

### 3.4 运营工具

| 脚本 | 说明 |
|------|------|
| `scripts/funnel-report.sh` | 读 API `/v1/analytics/funnel` 或本地 curl，输出转化率 |

### 3.5 测试

| 项 | 说明 |
|----|------|
| `backend/tests/test_analytics.py` | 事件写入、funnel 聚合、micro 反馈 |

## 五、验收标准

1. 走通一遍闭环后，`product_events` 中可按 `event_name` 查到 ≥6 种服务端事件
2. `GET /v1/analytics/funnel` 返回各步 count 与逐步转化率
3. 结果页 / 画像页可提交微反馈，`micro_feedback` 有记录
4. `./scripts/funnel-report.sh` 在内测环境可运行并输出可读报告
5. `pytest tests/test_analytics.py` 通过

---

## 六、内测数据与隐私

- 公开画像页仅记 `session_id` + `share_code`，不记完整 IP（可选 hashed）
- 内测数据存于测试库，文档已说明可能清空
- 微反馈 optional_text 限制 500 字
