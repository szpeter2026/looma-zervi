## 一、闭环定义

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  C端测完     │ →  │  生成分享码  │ →  │  查看画像    │ →  │  试用 T-space │
│  人格测试    │    │  裂变链接    │    │  公开/导入   │    │  Stub 升级   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
   PlanetX            referral API       SaaS 画像页         /pricing
```

**验收标准（内测可演示）**：

1. 用户在 PlanetX 完成 8 题测试，刷新页面后结果仍可展示
2. 结果页生成真实分享码，链接形如 `{T-space}/candidate/share/{CODE}`
3. 任何人打开链接可看到求职者人格画像（无需登录）
4. 查看者 **个人注册** T-space 并绑定企业后，可「导入候选人」并在列表中查看
5. 定价页可点击「开始 7 天试用」触发 Stub 升级至 pro

## 四、API 契约（闭环专用）

### 4.1 创建画像分享码

```
POST /v1/referral/create
Authorization: Bearer {jwt}
{ "purpose": "profile_share" }

→ 201 { "code": "A1B2C3D4", "purpose": "profile_share", "share_url": "..." }
```

### 4.2 公开查看画像（无需登录）

```
GET /v1/referral/profile-view/{code}

→ 200 {
  "personality_type": "星云艺术家",
  "personality_detail": { "name", "emoji", "tagline", "desc", "traits" },
  "user_display": "求职者",
  "xp": 50, "level": 1
}
```

### 4.3 企业代表导入候选人

```
POST /v1/enterprise/candidates/import-share
Authorization: Bearer {jwt}   # 已绑定企业的个人用户
{ "share_code": "A1B2C3D4" }

→ 201 { "id", "name", "profile_data", "user_id" }
```

### 4.4 Stub 试用升级

```
POST /v1/payment/upgrade
{ "tier": "pro" }

→ 200 { "tier": "pro", "message": "[STUB] ..." }
```
