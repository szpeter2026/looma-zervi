## 一、网站用途

**Looma Zervi** 是一个 AI 驱动的求职与职业发展平台，面向 Z 世代求职者与企业招聘方，提供智能化、游戏化的职业服务。**所有人统一个人注册**；企业能力通过自愿绑定的「企业代表」上下文开启（详见第二节）。

### 核心价值

| 服务对象 | 核心诉求 | 平台解法 |
|----------|---------|---------|
| **C 端求职者** | 职业方向迷茫、简历不会写、求职效率低 | 性格测试 → 画像分享 → AI 问答 → 职业探索 |
| **B 端企业侧** | 筛选成本高、简历量大、决策缺数据支撑 | 接住求职者主动分享的画像 → 职位匹配 → 简历解析 → 报告输出 |

### 多端覆盖

```
                ┌──────────────────────────┐
                │    Looma Zervi 后端       │
                │  Flask API（可容器化部署）  │
                │    统一鉴权 / 统一数据     │
                └──────┬───────────────────┘
           ┌───────────┼───────────┐
           ▼           ▼           ▼
     微信小程序     Web C 端     Web B 端
    (PlanetX 原生) (PlanetX Web) (T-space SaaS)
     求职者入口     求职者品牌站   企业工作台（T-space）
```

## 二、身份与企业商业模式

> **本节目的**：避免将 T-space 误解为「HR 专用账号体系」或传统 ATS 售卖模式。完整商业论证见 `reports/` 下 BP 与市场调研文档。

### 2.1 统一个人账号

- 所有人（求职者、招聘负责人、业务同事）均通过 **个人注册**（邮箱 / 微信）获得同一套用户身份。
- **不存在**「HR 专用账号」「企业开户账号」等第二种用户类型。
- PlanetX 与 T-space 是同一底座上的 **不同品牌与能力入口**，不是两套账号体系。

### 2.2 企业代表 = 可选、可撤销的上下文

类似企业微信的个人 / 企业身份关系：

```
个人注册（users）
    │
    ├─ 不绑定企业 → 纯 C 端：PlanetX 测试、问答、舰队等
    │
    └─ 自愿开启企业版 / 标注企业代表
           → 创建或加入企业（enterprise_users）
           → 获得 T-space 企业上下文：导入候选人、职位匹配、知识库等
           → 离职或不再代表企业 → 解绑企业标签（个人账号保留）
```

- **企业代表**：已绑定某企业的个人用户，在该企业上下文内操作；不是独立账号类型。
- 内测期可在 T-space `/candidates` 一键创建企业并绑定；正式产品将显式提供「开启企业版 / 解绑企业」流程（v0.1.0 完整离职解绑 API 待补）。

### 2.3 商业模式：C→B 画像引荐

与传统招聘 SaaS、典型内推 **均不相同**，但有可类比之处：

| 维度 | 传统招聘 SaaS | 典型内推 | **Looma / T-space** |
|------|--------------|---------|----------------------|
| 谁发起 | 企业发 JD、买曝光 | 在职员工推荐熟人 | **求职者**测完人格后 **主动** 分享画像 |
| 传递物 | 简历 / 平台流量 | 「我认识 TA」 | **经同意分享的结构化人格画像** |
| 账号 | HR 席位 / 企业租户 | 员工账号 | **统一个人账号 + 可选企业上下文** |
| C 端 | 多为平台库存 | 不参与 | **PlanetX 免费获客、叙事与裂变** |
| 企业付费点 | ATS 席位、职位发布 | 内推奖金（企业成本） | **接住人才流 + AI 工具（匹配 / 面试 / 知识库）** |

**核心闭环**（内测演示见第四节 4.6）：

```
PlanetX 测人格 → 生成画像分享码 → 公开页可查看（免登录）
    → 查看者个人注册 T-space → 可选绑定企业 → 导入候选人 → 试用 / 付费
```

这不是「企业先买系统再等简历」，而是 **C 端自生长 + 求职者主权下的画像引荐**——有内推的「定向触达」形态，但没有员工背书与内推奖金机制。

### 2.4 文档与界面用语约定

| 避免（易误解） | 推荐使用 |
|----------------|----------|
| HR 账号 / HR 注册 | **个人注册**；需要时写 **企业代表**（已绑定企业的个人用户） |
| HR 链接 / HR 专属 | **T-space 画像链接**（求职者分享给招聘方的公开链接） |
| B 端用户 | **处于企业上下文中的个人用户** |
| HR 看画像 | **查看者打开画像**（内测阶段常为未来企业代表；链接本身免登录） |

埋点事件 `hr_register_from_share` 等为历史命名，语义是「经画像分享链路进入 T-space 的个人注册」，**不表示 HR 专用账号**（见 [内测埋点与闭环漏斗方案.md](./内测埋点与闭环漏斗方案.md)）。

## 三、品牌定位

### 品牌架构

| 品牌 | 全称 | 定位 | 受众 |
|------|------|------|------|
| **Looma Zervi** | — | 平台母品牌 | 全域 |
| **PlanetX** | PlanetX | C 端求职者品牌 | 在校生 / 应届生 / 初级职场人 |
| **T-space** | T-space | B 端企业品牌 | 企业代表 / 招聘负责人 / 中小企业 |

### 品牌调性

- **PlanetX（C端）**：年轻化、游戏化、叙事化 — "探索你的职业星球"
- **T-space（B端）**：专业、高效、数据驱动 — "AI 赋能的智能招聘空间"

### 差异化卖点

1. **C→B 画像裂变** — 测完人格一键生成分享码，招聘方可免登录查看画像；企业代表注册后可导入候选人
2. **游戏化留存** — 舰队 / 任务体系驱动求职行为，告别枯燥填表
3. **叙事化品牌** — 第 1 幕 T-space 沉浸故事体验，区别于传统招聘平台
4. **AI 全链路** — 从性格洞察、智能问答、简历解析到职位匹配，关键环节 AI 加持
5. **跨端协同** — Web C/B 邮箱账号可互通；小程序与 Web 账号体系独立（内测期）
6. **自研可控** — 非 SaaS 套壳，核心能力自研，数据安全自主

### 4.2 微信小程序 — PlanetX

**定位**：求职者的随身 AI 职业助手

| 页面 | 功能简述 |
|------|---------|
| **闪屏页** | 启动加载，自动鉴权，静默登录；处理分享链接 `ref` 核销 |
| **母舰 Hub** | 游戏化主中心，任务进度、功能入口（含分享任务跳转结果页） |
| **问答** | AI 职业咨询；展示意图标签与 `intent_confidence` |
| **我的** | 个人中心，配额管理 |
| **答题** | 趣味性格测试问卷 |
| **结果** | 人格报告 + 分享码 / T-space 画像链接 + 微信分享 |
| **邮箱绑定** | 绑定邮箱账号（`pages/auth`） |

**技术参数**：原生 WXML/WXSS/TypeScript，基础库 3.15.0，AppID `wx6563950292092013`

**本地彩排**：微信开发者工具 → `packages/miniprogram`，勾选「不校验合法域名」，`utils/config.ts` 指向本地 API（默认 `http://127.0.0.1:5200`）。

### 4.3 Web C 端 — PlanetX Web

**定位**：求职者品牌体验站

| 路由 | 功能简述 |
|------|---------|
| `/` 主页 | 状态机：loading → auth → onboarding → hub → quiz → result |
| `/tspace` | T-space 第 1 幕沉浸式叙事体验 |
| `/quiz` | 性格测试（需登录） |
| `/result` | 测试结果报告 + SharePanel 分享码 / T-space 画像链接（需登录） |

**技术栈**：React 18 + TypeScript + Vite 5 + Zustand + React Router v6

### 4.4 Web B 端 — T-space SaaS

**定位**：企业 AI 招聘工作台（个人账号 + 企业上下文；共 **12** 条路由）

| 路由 | 功能简述 | 登录要求 |
|------|---------|---------|
| `/login` / `/register` | 邮箱注册登录 | 公开 |
| `/` 仪表盘 | API 健康、配额余量、快捷操作 | 公开 |
| `/pricing` 定价 | 套餐对比 +「开始 7 天试用」Stub 升级 | 公开 |
| `/candidate/share/:code` | **公开查看**求职者人格画像 | **免登录** |
| `/query` 智能问答 | AI 对话，知识库检索增强 | 需登录 |
| `/poetry` 诗词文库 | 5.8 万首古典诗词检索浏览 | 需登录 |
| `/jobs` 职位匹配 | 职位管理 + 匹配 + 页内「查征信」 | 需登录 |
| `/resume` 简历解析 | 上传简历 → AI 结构化提取 | 需登录 |
| `/reports` 报告中心 | 分析报告生成与管理 | 需登录 |
| `/candidates` 候选人 | 列表 + 从分享码导入候选人 | 需登录 |
| `/candidates/:id` | 候选人详情与画像数据 | 需登录 |

**技术栈**：React 18 + TypeScript + Vite 5 + Tailwind CSS 3.4 + Zustand

### 4.5 共享核心层

`@looma/shared-core` 是三个前端的共享内核，统一管理：

- **API 客户端**：`ApiClient` 类 + **13** 个领域 API 工厂（auth、chat、game、enterprise、referral、resume、jobs、reports、credit、narrative、act1、payment、quota）
- **类型定义**：auth、chat、game、resume、enterprise、referral 等全域 TypeScript 类型
- **常量与验证**：配额限制、品牌标识、路由字典、输入校验
- **存储适配器**：`webStorageAdapter` / `wxStorageAdapter` 双端存储抽象

### 4.6 最小闭环（内测演示主线）

v0.1.0 核心演示路径，详见 [最小闭环执行计划.md](./最小闭环执行计划.md)：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  C端测完     │ →  │  生成分享码  │ →  │  查看画像    │ →  │  试用 T-space │
│  人格测试    │    │  裂变链接    │    │  公开/导入   │    │  Stub 升级   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
   PlanetX            referral API       SaaS 画像页         /pricing
```

**验收要点**：

1. PlanetX（Web 或小程序）完成测试，刷新后结果仍可展示
2. 结果页生成真实分享码，链接形如 `{T-space}/candidate/share/{CODE}`
3. 任何人打开链接可见人格画像（无需登录）
4. 查看者 **个人注册** T-space 后，绑定企业（内测可一键创建）并「导入候选人」，在 `/candidates` 查看
5. `/pricing` 点击「开始 7 天试用」触发 Stub 升级至 pro

**自动化验证**：`scripts/verify-closed-loop.sh`（闭环 API）、`scripts/rehearsal-local.sh`（闭环 + 持久化 + RAG）。

### 6.3 内测流程

#### 轨 A — 最小闭环演示（推荐）

```

   → PlanetX Web (:5173) 或小程序完成答题
   → 结果页获取分享码 / T-space 画像链接

第 2 步：查看画像（免登录）
   → 打开 {T-space}/candidate/share/{CODE}
   → 确认人格类型、特质、详情展示正常

第 3 步：个人注册并导入（企业代表上下文）
   → T-space → /register 个人邮箱注册（非「HR 开户」）
   → /candidates → 创建 / 绑定企业 → 从分享码导入候选人

第 4 步：试用升级
   → /pricing → 点击「开始 7 天试用」
   → 仪表盘确认 tier 升级为 pro、配额变化

   → 提交 Bug / 建议，附复现步骤 + 截图
```

```
第 1 步：个人注册 → T-space /register → 进入仪表盘
```

### 6.4 已知限制（内测阶段）

| 限制 | 说明 | 预计解决 |
|------|------|---------|
| 小程序真机 / 审核 | 需 HTTPS + 微信合法域名；本地开发者工具可彩排 | 备案通过后配置 |
| 无 HTTPS（公网） | 内测 IP 暂用 HTTP | 备案后 Nginx + SSL |
| 支付为 Stub | 定价页可试用升级，**无真实微信支付** | Phase 5 |
| 通用 RAG 知识库 | 诗词 5.8 万首可用；非诗词类 `/v1/ask` 的 `sources` 可能为空 | 后续导入文档 |
| 诗词 Ask 响应 | `poetry` 意图常返回 `extracted`，`answer` 可能为空 | 前端已适配 / 后端优化 |
| 邮件服务未配置 | 密码重置等需邮件 | 后续迭代 |
| 跨端账号 | Web C/B 邮箱可互通；小程序与 Web 账号独立 | 后续统一 |

### 6.5 注意事项

1. **数据安全**：内测数据存于测试库，正式上线前可能清空
2. **稳定性**：后端为单节点部署，可能出现偶发重启
3. **API 限流**：未配置生产级限流，请勿压力测试
4. **浏览器兼容**：推荐 Chrome / Edge 最新版，不支持 IE
5. **验收工具**：部署后可运行 `scripts/verify-deployment.sh <IP或域名>`；本地可运行 `scripts/rehearsal-local.sh`
6. **漏斗与埋点**：走通闭环后运行 `scripts/funnel-report.sh` 或 `GET /v1/analytics/funnel`；方案见 [内测埋点与闭环漏斗方案.md](./内测埋点与闭环漏斗方案.md)
