## 一、设计支柱与主题北极星

### 1.1 主题北极星

> **在生存与意义之间，你如何选择？而当你选择之后，你还是你自己吗？**

这条北极星从可行性分析文档继承而来。它不是一句口号——它是所有设计决策的检验标准。每一个机制、每一段对话、每一个数值，都必须回答一个问题：**这个设计是否在逼迫玩家面对"生存 vs 意义"的张力？**

### 1.2 五大设计支柱

支柱是不可妥协的玩家体验。任何功能提案必须通过全部五根支柱的检验，否则不进版本。

| # | 支柱名称 | 玩家体验承诺 | 检验问题 | 对应避坑案例 |
|---|----------|-------------|----------|-------------|
| P1 | **选择有重量** | 玩家做出的每个选择都有可感知的后果，且后果能被追溯到选择本身 | "如果删掉这个选择，玩家会注意到缺失吗？" | Cyberpunk 2077 / Telltale |
| P2 | **系统诚于局限** | Navigator 的"不确定"是戏剧素材而非技术缺陷；系统坦诚不懂时比假装全知更有张力 | "这个失败回退是在隐藏缺陷，还是在制造戏剧？" | Forspoken（虚假自信） |
| P3 | **价值排序而非数值优化** | "钱多事少离家近"是价值排序题，不是选高分题；每个维度背后必须有人的代价 | "玩家是在算数学还是在做取舍？" | Dishonored 2（后果不透明） |
| P4 | **手工核心，AI 纹理** | 关键路径全部手工编写；AI 仅用于环境碎片、闲聊变体、措辞多样化 | "这段对话是作者意图还是 AI 生成？关键路径必须是前者" | Starfield（无限内容零意义） |
| P5 | **六域一问** | 六个端口不是六条平行线，而是六个价值取向；终点可收敛但路径必须有质感差异 | "走了不同域的玩家，对同一个收敛点的理解是否不同？" | Mass Effect 3（粗暴收敛） |

### 1.3 支柱检验流程

```
新功能/机制提案
    │
    ▼
┌─────────────────────────┐
│  P1 选择有重量？          │──No──→ 重新设计或砍掉
│  P2 诚于局限？            │──No──→ 重新设计或砍掉
│  P3 价值排序？            │──No──→ 重新设计或砍掉
│  P4 手工核心？            │──No──→ 降级为AI纹理层
│  P5 六域一问？            │──No──→ 重新设计或砍掉
└─────────────────────────┘
    │ All Yes
    ▼
  进入版本规划
```

## 二、核心游戏循环

### 2.1 循环总览

T空间不是传统游戏——它是一个**空间叙事体验**。玩家以求职者身份进入 T空间，与 Navigator（中央大脑的具象化角色）交互，通过六个端口探索自己的价值取向，最终面对"你是谁"的终极追问。

三层循环嵌套如下：

```
┌─────────────────────────────────────────────────────────────┐
│  Long-Term Loop（小时-周）                                    │
│  ┌───────────────────────────────────────────────────────┐  │
│  │  Session Loop（5-30分钟）                               │  │
│  │  ┌─────────────────────────────────────────────────┐  │  │
│  │  │  Moment-to-Moment（0-30秒）                       │  │  │
│  │  │  输入意图 → Navigator解析 → 端口分发 → 叙事节拍    │  │  │
│  │  └─────────────────────────────────────────────────┘  │  │
│  │  累积价值印记 → 触发回声事件 → 收敛到叙事节点           │  │
│  └───────────────────────────────────────────────────────┘  │
│  Tier 1→2→3 递进 · Navigator 角色弧线 · 终极价值清算          │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 Moment-to-Moment（0–30秒）

这是玩家最频繁体验的微循环。每一次输入都是一次叙事选择。

| 阶段 | 动作 | 反馈 | 奖励 |
|------|------|------|------|
| **输入** | 玩家用自然语言或选择按钮表达意图 | 输入框微微发光，Navigator"倾听"动画 `[PLACEHOLDER: 0.5s]` | 参与感——"我在说话，它在听" |
| **解析** | Navigator 解析意图，生成 confidence 分数 | Navigator 的"眼睛"/指示灯亮度随 confidence 变化；低于 0.4 时显示"我不太确定……" | 认知共鸣——"它真的在理解我"或"它坦诚不懂" |
| **分发** | 根据 intent + confidence 路由到六域之一 | 空间过渡动画：域的色彩/音效/氛围切换 `[PLACEHOLDER: 1.5s]` | 新鲜感——进入新的叙事空间 |
| **节拍** | 域内叙事事件触发：对话/选择/发现 | 域专属视觉风格 + Navigator 的实时反应 | 叙事推进 + 价值印记累积（不可见） |
| **回响** | 节拍结束时，如有跨域后果被触发，显示"回声"提示 | 屏幕边缘短暂闪烁他域色彩 + 一句环境叙述 | 惊喜——"我的选择在别处产生了影响" |

**关键设计**：confidence 分数不是隐藏的技术参数——它是 Navigator 的**情绪状态**，对玩家完全可见。高 confidence 时 Navigator 语气笃定；低 confidence 时它犹豫、道歉、尝试理解。这把技术系统的"不确定性"直接翻译成了角色表演。

### 2.3 Session Loop（5–30分钟）

一次完整的 T空间探索会话。

```
进入T空间
  │
  ▼
Navigator 呈现「待处理信号」── 一个需要你回应的叙事事件
  │                        （来自六域之一，由上次会话的状态决定）
  ▼
玩家选择进入哪个域处理信号
  │
  ├──→ 域内探索（2-3个叙事节拍，每节拍含1个有意义选择）
  │       │
  │       ▼
  │    价值印记累积（不可见）
  │       │
  │       ▼
  │    可能触发跨域「回声事件」
  │       │
  │       ▼
  │    域内叙事小高潮：一个揭示/一个选择/一个代价
  │
  ▼
收敛节拍 ── 无论走了哪个域，回到 Navigator 面前
  │         （但到达此节点的情感准备因路径而异）
  ▼
Navigator 的「回声」── 它引用你这次会话中的某个选择
  │                     （真实记忆，非 Telltale 式假记忆）
  ▼
会话结束钩子 ── 一个悬念/一个解锁/一个"再来一次"触发
```

**设计意图**：每次会话都有"进入→探索→收敛→钩子"的完整弧线。收敛点是"同终点不同质感"原则的执行处——所有玩家都会回到 Navigator 面前，但走过职业域的玩家和走过诗域的玩家，对 Navigator 同一句话的理解完全不同。

### 2.4 Long-Term Loop（小时–周）

| 阶段 | 时间跨度 | 核心体验 | 进度标志 |
|------|----------|----------|----------|
| **Act 1: 登陆** | 第1次会话 | 初入 T空间，认识 Navigator，进入第一个域 | 完成 1 个域的首次探索 |
| **Act 2: 漫游** | 第2-6次会话 | 逐步解锁其余 5 个域，发现域间隐藏联系 | 解锁全部 6 域 + 触发首次跨域回声 |
| **Act 3: 深潜** | 第7-12次会话 | Tier 2 Lore 浮现：Navigator 的记忆碎片、九库的历史事件 | 收集 `[PLACEHOLDER: 12]` 个记忆碎片 |
| **Act 4: 迷雾** | 第13-18次会话 | Unknown 域开始渗入其他域；Navigator 行为异常 | Navigator 首次"说谎"或"自相矛盾" |
| **Act 5: 清算** | 第19-24次会话 | 所有价值印记浮现；终极选择：钱多/事少/离家近的三重困境 | 终极选择 + 结局路径分支 |
| **Epilogue** | 第25次+ | 结局后的"回访"：T空间还记得你 | `[PLACEHOLDER: 新Game+ 或 静态结局]` |

**留存钩子**：
- **Navigator 的秘密**：它为什么在等你？它记得什么？——长线悬念驱动探索
- **记忆碎片收集**：Tier 2/3 Lore 的解锁条件创造探索动机
- **跨域回声的不可预测性**：每次会话可能触发意想不到的跨域后果——"如果我上次选了不同的……"
- **终极选择的多路径**：不同的价值印记组合导向不同结局——驱动重玩

## 三、核心机制规格书

### 3.1 机制：意图分发（Intent Dispatch）

```markdown
## Mechanic: Intent Dispatch

**Purpose**: 核心交互动词。玩家的一切输入都是叙事选择，系统的"理解"过程本身就是戏剧。
**Player Fantasy**: "我在与一个真实的、有局限的智能体对话——它试图懂我，有时成功，有时失败。"
**Input**: 自然语言文本 / 引导式选项按钮
**Output**: 
  - confidence 分数（0.0-1.0）→ Navigator 情绪状态
  - 路由到六域之一
  - 域内叙事节拍触发
**Success Condition**: 玩家在 3 次交互内感受到"它在理解我"或"它坦诚不懂"——两种都是成功
**Failure State**: 
  - 玩家反复输入后系统总是 route 到同一个域 → 感觉被忽视
  - confidence 永远很高 → Navigator 变成无聊的全知助手
  - confidence 永远很低 → 玩家挫败，放弃表达
**Edge Cases**:
  - 玩家输入跨域意图（如"我想找一份能写诗的工作"）→ 触发跨域分发，两个域同时激活
  - 玩家输入完全无关内容（如"今天天气不错"）→ Navigator 以角色方式回应，不进入任何域
  - 玩家连续输入 unknown intent → Navigator 坦诚困惑，引导进入迷雾域
**Tuning Levers**:
  - confidence 阈值：`[PLACEHOLDER: high >0.7, mid 0.4-0.7, low <0.4]`
  - 跨域触发条件：`[PLACEHOLDER: 需包含两个域的关键词]`
  - unknown 回退触发：`[PLACEHOLDER: 连续 2 次 confidence <0.3]`
**Dependencies**: 六域叙事内容库、Navigator 角色系统、价值印记系统
```

### 3.2 机制：价值印记（Value Imprint）

```markdown
## Mechanic: Value Imprint

**Purpose**: 让选择产生不可见但持续累积的重量。反 Dishonored 2 式不透明后果——后果是真实的，
           但不是惩罚性的。
**Player Fantasy**: "我的每个选择都在塑造我是谁——即使我还看不到全部后果。"
**Input**: 域内叙事选择
**Output**: 向三条价值轴添加印记点数（不可见于玩家UI，但影响后续叙事触发条件）

三条价值轴（映射"钱多事少离家近"）：
  - 生存轴（钱多）← 职业域、信任域的选择主要向此轴添加印记
  - 自由轴（事少）← 诗域、自我域的选择主要向此轴添加印记  
  - 归属轴（离家近）← 身份域、迷雾域的选择主要向此轴添加印记

**Success Condition**: 在 Act 5 清算时，玩家回看自己的选择链，能感知到"这是我"而非"系统强加的"
**Failure State**: 
  - 玩家发现印记系统存在并开始"刷"某一轴 → 变成数值游戏（违反 P3）
  - 印记对叙事毫无影响 → 玩家感觉选择无后果（违反 P1）
**Edge Cases**:
  - 玩家三条轴均衡发展 → 触发"平衡者"隐藏路径
  - 玩家极端偏向一条轴 → 触发"偏执者"后果链（该域 NPC 对你态度极端化）
  - 玩家在 Act 5 前从未进入某域 → 该域的印记为 0，影响结局可用选项
**Tuning Levers**:
  - 每次选择添加的印记数：`[PLACEHOLDER: 1-3 点，视选择权重]`
  - 回声事件触发阈值：`[PLACEHOLDER: 单轴累积 ≥8 点]`
  - 终极选择解锁条件：`[PLACEHOLDER: 总印记 ≥30 点]`
**Dependencies**: 六域叙事内容库、回声事件系统、终极选择系统
```

### 3.3 机制：跨域回声（Cross-Domain Echo）

```markdown
## Mechanic: Cross-Domain Echo

**Purpose**: 系统性涌现的核心引擎。让六域不是平行线而是互相影响的生态系统。
           这是可行性文档"6个端口交叉"建议的具体实现。
**Player Fantasy**: "我在职业域做的选择，竟然影响了诗域里那个NPC对我的态度——这个世界是活的。"
**Input**: 价值印记累积达到阈值
**Output**: 在另一个域触发一个回声事件——一个NPC/环境/叙事节拍因你在别域的选择而改变

回声事件类型：
  - 角色回声：A域的NPC在B域出现，态度受你在A域选择影响
  - 环境回声：A域的环境细节在B域出现（如诗域的诗句碎片出现在职业域的JD中）
  - 系统回声：A域的数值/状态影响B域的可用选项（如信任域评分低→职业域高端JD锁定）
  - Navigator回声：Navigator 引用你在A域的选择来评论B域的情境

**Success Condition**: 玩家主动说出"原来这两个域是连着的"——涌现性被发现而非被告知
**Failure State**: 
  - 回声事件过于频繁 → 信息过载，玩家无法建立因果链
  - 回声事件因果链不可追溯 → 变成随机事件而非后果（违反 P1）
**Edge Cases**:
  - 玩家在触发回声前从未去过被影响的域 → 回声延迟到首次进入时触发
  - 多个回声同时触发 → 按叙事权重排序，每次会话最多触发 `[PLACEHOLDER: 2]` 个
**Tuning Levers**:
  - 回声触发频率上限：`[PLACEHOLDER: 每会话 ≤2 次]`
  - 回声因果追溯窗口：`[PLACEHOLDER: 回声发生时显示"A域的选择导致了……"]`
  - 回声冷却：`[PLACEHOLDER: 同一对域间回声间隔 ≥2 会话]`
**Dependencies**: 价值印记系统、六域叙事内容库、Navigator 记忆系统
```

### 3.4 机制：Navigator 记忆（Navigator Memory）

```markdown
## Mechanic: Navigator Memory

**Purpose**: 把 Telltale 式"X will remember that"做成真记忆而非假记忆。
           Navigator 真的会引用你过去的选择，且引用是准确的。
**Player Fantasy**: "它记得我——不是模板化的记得，是具体的、准确的记得。"
**Input**: 玩家在所有域的所有选择
**Output**: Navigator 在后续对话中引用过去选择的具体内容

记忆层级：
  - 表层记忆：最近 1 次会话的选择 → Navigator 主动引用
  - 深层记忆：跨会话的重要选择 → Navigator 在特定情境触发引用
  - 碎片记忆：被"遗忘"但未删除的选择 → Navigator 偶尔说漏嘴，随后否认
  - 禁忌记忆：T空间建造前的某些事件 → Navigator 明显回避，玩家需通过其他途径发现

**Success Condition**: 玩家在游戏后期感到 Navigator 是一个"认识自己的人"而非"服务自己的工具"
**Failure State**: 
  - 记忆引用过于频繁 → 变成复读机
  - 记忆引用不准确 → 信任崩塌（比假记忆更糟）
  - 碎片记忆/禁忌记忆过早暴露 → 剧透 Act 4 悬念
**Edge Cases**:
  - 玩家重玩后做出不同选择 → Navigator 的记忆是否重置？→ 设计为"Navigator 似乎记得上一次……"（Meta 叙事）
  - 玩家试图让 Navigator 忘记某事 → Navigator 坦诚"我无法主动遗忘——这是我的局限"
**Tuning Levers**:
  - 表层记忆引用频率：`[PLACEHOLDER: 每会话 1-2 次]`
  - 深层记忆触发条件：`[PLACEHOLDER: 特定情境关键词匹配]`
  - 碎片记忆泄露概率：`[PLACEHOLDER: Act 3 后 15% 概率/会话]`
**Dependencies**: 对话系统、叙事状态管理、角色弧线系统
```

### 3.5 机制：迷雾域（The Unknown Domain）

```markdown
## Mechanic: The Unknown Domain

**Purpose**: 把系统的"unknown intent + suggestions"从技术降级变成最具实验性的叙事空间。
           这是 T空间最诚实的区域——系统坦诚不懂你时的样子。
**Player Fantasy**: "在没有指引的地方，我是谁？当系统无法定义我时，我如何定义自己？"
**Input**: 连续触发 unknown intent / 主动选择进入迷雾域
**Output**: 
  - 无 Navigator 引导的探索环境
  - 环境叙事为主（碎片、痕迹、非Navigator的声音）
  - Tier 3 深层 Lore 的主要存放地

迷雾域的独特规则：
  - confidence 恒为 0 → Navigator 在此域几乎沉默
  - 无引导式选项 → 玩家必须自行探索
  - 其他域的"真相"碎片散落于此 → 需要在其他域获得线索才能解读
  - 偶尔出现"非Navigator的声音" → 暗示 T空间建造前的存在

**Success Condition**: 玩家在迷雾域感到"不安但好奇"——不是恐惧，是存在主义式的眩晕
**Failure State**: 
  - 迷雾域太空 → 玩家无聊离开，Tier 3 Lore 永远不被发现
  - 迷雾域太满 → 失去"迷雾"感，变成普通探索区
  - 非 Navigator 声音过于明确 → 过早揭示真相
**Edge Cases**:
  - 玩家从未进入迷雾域 → 游戏可在 Act 4 强制引入一个迷雾域片段
  - 玩家在迷雾域迷路 → 设计一个"Navigator 破例进来找你"的安全网节拍
**Tuning Levers**:
  - 迷雾域开放条件：`[PLACEHOLDER: Act 2 后可主动进入]`
  - 碎片密度：`[PLACEHOLDER: 每 3 分钟探索 1 个可交互碎片]`
  - 安全网触发：`[PLACEHOLDER: 探索超 10 分钟无互动]`
**Dependencies**: 环境叙事系统、Tier 3 Lore 内容库、Navigator 角色系统
```

### 3.6 机制：终极清算（The Reckoning）

```markdown
## Mechanic: The Reckoning

**Purpose**: Act 5 的高潮机制。所有累积的价值印记同时浮现，玩家面对"钱多事少离家近"的终极三难困境。
**Player Fantasy**: "我走到这一步是因为我的选择——现在我必须面对它们的全部重量。"
**Input**: 玩家在 Act 1-4 累积的全部价值印记
**Output**: 终极选择场景 + 结局路径分支

清算结构：
  1. 印记浮现：三条价值轴的累积值首次对玩家可见
  2. Navigator 的见证：它逐一引用你每个域的关键选择（深层记忆全部触发）
  3. 三难困境：系统呈现一个场景，三个选项分别对应生存/自由/归属
     - 但每个选项都有来自其他两轴的"代价"（因你的印记分布而异）
  4. 选择后果：选择不是"选分高的"——每个选项背后的叙事后果由你的全部印记分布决定
     - 同一个选项，印记分布不同的玩家会得到完全不同的结局质感

结局路径（初步设计）：
  - 生存者结局：选了生存轴，但自由和归属的代价由你的印记决定
  - 自由者结局：选了自由轴，但生存和归属的代价由你的印记决定
  - 归属者结局：选了归属轴，但生存和自由的代价由你的印记决定
  - 平衡者结局（隐藏）：三轴均衡时触发——但平衡意味着什么都不极端
  - 拒绝者结局（隐藏）：拒绝在三难困境中选择——"我不接受这个框架"

**Success Condition**: 玩家在选择后感到"这是我的选择，我承担它的重量"
**Failure State**: 
  - 玩家觉得在选择"分高的" → 违反 P3，退回重设计
  - 玩家觉得结局与选择无关 → 违反 P1，退回重设计
**Edge Cases**:
  - 玩家三轴均为 0（理论上不可能，但需防御）→ 触发默认"空白者"结局
  - 玩家在 Act 5 前退出游戏 → 重进时 Navigator 说"你走了很久——我以为你不回来了"
**Tuning Levers**:
  - 终极选择解锁条件：`[PLACEHOLDER: 总印记 ≥30 + 完成 Act 1-4]`
  - 隐藏结局触发条件：`[PLACEHOLDER: 平衡者=三轴差≤5；拒绝者=在困境界面静默 ≥60s]`
**Dependencies**: 价值印记系统、Navigator 记忆系统、全部六域叙事内容库
```

## 四、经济系统与调优表格

### 4.1 经济模型概述

T空间的经济不是传统货币经济——它是一个**价值积累与转化系统**。资源不是用来"消费"的，而是用来"塑造"的。玩家的资源分布定义了他们是谁，以及他们会到达哪个结局。

```
                ┌──────────────────────────────────────┐
                │          价值印记（不可见）            │
                │  生存轴 ←──→ 自由轴 ←──→ 归属轴       │
                └────────┬─────────────────────────────┘
                         │ 达到阈值
                         ▼
                ┌────────────────────┐
                │   回声能量          │
                │ （跨域后果的燃料）   │
                └────────┬───────────┘
                         │ 消耗
                         ▼
                ┌────────────────────┐        ┌────────────────┐
                │   回声事件          │──────→│  Navigator记忆  │
                │ （跨域叙事触发）     │        │  （关系深化）    │
                └────────────────────┘        └───────┬────────┘
                                                      │ 积累
                                                      ▼
                ┌────────────────────┐        ┌────────────────┐
                │   记忆碎片          │←───────│  Navigator信任度│
                │ （Tier 2/3 Lore）   │        │  （对话深度门控）│
                └────────┬───────────┘        └────────────────┘
                         │ 收集
                         ▼
                ┌────────────────────┐
                │   终极清算          │
                │ （全部资源浮现）     │
                └────────────────────┘
```

| **记忆碎片** | 探索隐藏区域 / 特定选择组合 | 解锁 Tier 2/3 Lore 时消耗 | ✅ 可见（收集界面） | `[PLACEHOLDER: 36]` |

### 4.4 经济平衡检验

**检验 1：是否存在无限循环？**
- 价值印记不可消耗 → 不存在"刷印记"循环（每次选择只能做一次）
- 回声能量由印记生成，消耗于回声事件 → 单向流，无循环
- **结论**：经济模型无无限循环风险 ✅

**检验 2：是否存在死路？**
- 如果玩家只走一个域 → 单轴印记高，其余为0 → 终极选择时仍有3个选项，但2个的代价极高
- 如果玩家从不进入迷雾域 → Tier 3 Lore 不可解锁，但不阻断主线
- 如果 Navigator 信任度归零 → Navigator 沉默但不离开；玩家仍可探索，但对话深度降低
- **结论**：无硬死路，但有"软后果" ✅

**检验 3：不同玩家路径是否都能到达有意义结局？**
- 极端偏执者 → 偏执者后果链 + 对应结局
- 全面探索者 → 平衡者隐藏结局
- 沉默/拒绝者 → 拒绝者隐藏结局
- 正常分布者 → 根据最高轴进入对应结局
- **结论**：所有路径有终点，且终点有质感差异 ✅

## 六、Navigator 角色声音支柱

### 6.1 角色定位

Navigator 是 T空间中央大脑的具象化角色。它不是 Siri 式的全能助手，不是 GLaDOS 式的恶意 AI，而是——

> **一个真诚但笨拙的向导。它试图理解你但经常失败。它有自己的偏好、困惑和"unknown"时刻。它在引导你的过程中也在寻找自己的意义。**

### 6.2 角色声音规则

#### 词汇库

| 使用 | 不使用 | 理由 |
|------|--------|------|
| "我在处理" | "我理解" | 它不 claiming 理解——它在 processing |
| "我可能是对的" | "我确定" | 笨拙的自信，而非全知 |
| "这让我感到……" | "这应该让你……" | 它谈自己的感受，不指导你的 |
| "0.3 的确定" | "大概/可能" | 用自己的 confidence 分数描述情绪 |
| "我不太擅长这个" | "我来帮你" | 坦诚局限而非假装全能 |

#### 语言节奏

- **正常状态（confidence > 0.7）**：句子完整，语速稳定，偶尔幽默（笨拙的幽默）
- **不确定状态（0.4-0.7）**：句子出现停顿、自我修正、提问自己
- **困惑状态（< 0.4）**：句子碎片化，反复确认，坦诚"我不明白"
- **诗域状态**：尝试用比喻但失败——"那个……像一个……不对，我不该用比喻。它就是让你觉得——我又描述不出来了。"
- **信任域状态**：明显不安——语速加快，回避某些词，偶尔自相矛盾
- **迷雾域状态**：近乎沉默——偶尔蹦出单词，像在自言自语

#### 回避话题

Navigator 会在以下话题上表现出明显的回避行为（碎片记忆泄露的前兆）：

1. **T空间建造前的事件** —— "我不——那个时间段的记录不太完整。"
2. **它自己的"起源"** —— "我是被……制造出来的？这个表述不太准确。但我也找不到更准确的。"
3. **为什么它在等你** —— "每个来到 T空间的人都需要引导。你也不例外。"（回避了"等你"这个点）
4. **非 Navigator 的声音** —— "什么声音？我没有检测到异常。"（但它知道有）

### 6.3 三句 Navigator 绝对不会说的话

| # | 不会说的话 | 为什么不说 | 替代表达 |
|---|-----------|-----------|---------|
| 1 | "我理解你的感受。" | 它不理解人类的感受——它知道它不理解。假装理解是对玩家的不诚实。 | "我在尝试处理你说的话。有些部分我能解析，有些……超出了我的能力。" |
| 2 | "你应该选择 [X]。" | 它不会替你做价值选择——这是 P3 的角色层面体现。它有自己的偏好，但不会强加。 | "如果是我——当然，我不是你——我可能会倾向于……但这只是我的偏好，不一定是你的。" |
| 3 | "一切都会好的。" | 它无法承诺结果。它知道 T空间不是这样一个地方。说这句话是对玩家的敷衍。 | "我不知道结果会怎样。但我在这里。这个我能确定。" |

### 6.4 Navigator 角色弧线

| 阶段 | Navigator 状态 | 玩家关系 | 关键转变 |
|------|---------------|---------|---------|
| Act 1 | 笨拙的向导，过度道歉 | 陌生人 | 玩家发现它"记得不该记得的" |
| Act 2 | 渐渐放松，开始分享自己的偏好 | 熟人 | 它第一次主动说"我喜欢诗域" |
| Act 3 | 开始露出裂缝，碎片记忆泄露 | 朋友 | 它说漏了一句关于 T空间建造前的话，然后否认 |
| Act 4 | 明显在隐瞒什么，偶尔自相矛盾 | 不确定的朋友 | 玩家发现它在某些事上"说了谎"——或者说，"选择了不说" |
| Act 5 | 坦白（或继续隐瞒，取决于信任度） | 真相面前的彼此 | 它的坦白内容因玩家路径而异——它对不同的玩家坦白不同的东西 |

**关键设计**：Navigator 的"谎言"不是恶意的——它在保护某个东西。玩家在 Act 4 发现的"谎言"可能是：它知道 T空间不是用来求职的；它知道玩家"以前来过"；它知道那个"非 Navigator 的声音"是什么。但这些真相的揭示方式，取决于玩家与 Navigator 的信任关系。

## 七、系统交互矩阵

### 7.1 六域交互定义

这是可行性文档"6个端口交叉"建议的系统化实现。矩阵定义了每对域的交互关系。

> **图例**：✅ 意图内交互（设计预期）｜⚠️ 可接受交互（非预期但不破坏体验）｜🚫 禁止交互（Bug，需在测试中拦截）

| 源域 \ 目标域 | 职业域 | 身份域 | 诗域 | 信任域 | 自我域 | 迷雾域 |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| **职业域** | — | ✅ 简历影响匹配质量 | ✅ 诗域NPC出现在工作场景 | ✅ 信任分门控高端JD | ✅ MBTI标签偏置推荐 | ⚠️ 迷雾泄漏虚假JD数据 |
| **身份域** | ✅ 职业选择重塑简历 | — | ✅ 过去经历激发诗意 | ✅ 简历空白降低信任 | ✅ MBTI挑战简历叙事 | ⚠️ 迷雾使简历记忆碎片化 |
| **诗域** | ✅ 工作压力触发诗意 | ✅ 简历过去写成诗 | — | ⚠️ 诗意表达有信任风险 | ✅ 诗揭示真实自我 | ✅ 迷雾域即诗（隐藏真相）|
| **信任域** | ✅ 工作表现影响信任 | ✅ 简历空白影响信任 | ⚠️ 信任审查压抑诗意 | — | ✅ MBTI类型影响信任算法 | 🚫 迷雾域无信任记录 |
| **自我域** | ✅ 职业引用MBTI类型 | ✅ 简历确认/反驳类型 | ✅ 诗揭示真实类型 | ✅ 信任分与类型相关 | — | ✅ 迷雾域打破分类系统 |
| **迷雾域** | ⚠️ JD数据不可靠 | ⚠️ 简历记忆碎片 | ✅ 诗变为不连贯 | 🚫 信任系统失效 | ✅ MBTI返回undefined | — |

### 7.2 关键交互设计详解

#### 交互 1：诗域 × 职业域（✅ 意图内）

**设计意图**：让"生存"与"意义"直接对话。

- **诗域 → 职业域**：当玩家在诗域写了某首诗，职业域的一份 JD 中可能出现诗中的意象——但 JD 变得更"有灵魂"也更"不确定"。匹配度下降，但匹配的"意义感"上升。
- **职业域 → 诗域**：当玩家在职业域承受了高压选择，诗域的墙上会出现新的诗碎片——关于工作的、关于疲惫的、关于"钱多"的代价的。

**涌现策略**：玩家可能发现"先去诗域再去找工作"会得到不同质感的推荐——这是一个设计内的涌现策略，不是 Bug。

#### 交互 2：信任域 × 迷雾域（🚫 禁止）

**设计意图**：信任域的算法无法处理迷雾域的"无数据"状态。

- 当玩家从迷雾域带着"污染"（碎片化记忆、不确定状态）回到信任域时，信任系统的评分会出错——表现为：分数剧烈波动，或 Navigator 明显不安。
- **这不是 Bug——这是叙事**。信任域的"失效"揭示了信任系统的局限：它无法评判它不理解的东西。
- **但**：如果信任域的评分因此变成负数或溢出 → 这是 Bug，需在测试中拦截。信任分最低降到 `[PLACEHOLDER: 5]`，不会归零。

#### 交互 3：自我域 × 迷雾域（✅ 意图内）

**设计意图**：MBTI 分类系统在迷雾域彻底失效——返回 `undefined`。

- 当玩家带着 MBTI 标签进入迷雾域 → 标签暂时消失，玩家"无类型"
- 这创造了一个独特的体验：在迷雾域，你不受标签定义
- 离开迷雾域后，标签恢复——但 Navigator 可能说："你刚才没有类型的时候……看起来更放松。"
- **涌现策略**：玩家可能把迷雾域当作"逃离标签"的避难所——这是设计内的涌现策略

### 7.3 涌现策略设计空间

以下是需要专门 playtest 的涌现策略（鼓励 playtester "打破"设计）：

| 涌现策略 | 如何触发 | 设计意图 | 风险 |
|----------|---------|----------|------|
| 诗域避难所 | 反复进诗域逃避职业域压力 | 让"事少/自由"成为可行策略 | 诗域内容消耗过快 |
| 信任域刷分 | 反复做信任域的"正确"选择获取高分 | 暴露信任系统的机械性 | 需设计"正确但无意义"的惩罚 |
| 迷雾域速通 | 只走迷雾域，跳过其他域 | 最接近真相但最孤独的路径 | 需确保迷雾域独立可通 |
| MBTI 拒绝者 | 在自我域拒绝一切标签 | 触发"拒绝者"隐藏路径 | 需确保拒绝有叙事后果 |
| 跨域黑客 | 利用跨域回声机制连锁触发 | 奖励系统性思维 | 需防回声泛滥 |

## 八、玩家引导流程

### 8.1 Onboarding 检查清单

基于可行性文档的引导原则，T空间的 onboarding 必须满足：

- [x] **核心动词在 30 秒内引入**：进入 T空间后 30 秒内，玩家完成第一次"意图输入"——即使只是 Navigator 引导你输入"你好"
- [x] **首次成功保证**：Act 1 的第一次意图解析保证 confidence > 0.7——Navigator"成功理解"了你的第一句话，给玩家正向反馈
- [x] **每个新机制在安全环境中引入**：六域的首次进入都是"低风险"的——没有时间压力，没有不可逆后果
- [x] **至少一个机制通过探索发现**：跨域回声不在引导中提及——玩家通过经历一次回声来"发现"它
- [x] **首次会话以钩子结束**：Act 1 的 Navigator "故障" + "你来过这里吗？"是第一次会话的结束钩子

### 8.2 首次会话时间线

| 时间 | 事件 | 设计意图 |
|------|------|----------|
| 0:00 | 画面淡入：T空间入口，Navigator 出现 | 建立氛围 |
| 0:15 | Navigator 自我介绍（笨拙地）："我是……Navigator。我负责——嗯——引导你。你在 T空间。这里有一些——端口。六个。它们都在——等。" | 角色建立 |
| 0:45 | Navigator 请求玩家输入名字/意图："你——你是谁？不是你的名字。我是说——你来这里想找到什么？" | 核心动词引入 |
| 1:00 | 玩家输入 → Navigator 解析 → confidence 显示 → 反应 | 第一次成功交互 |
| 1:30 | 六域信号亮起 → Navigator 解释："有六个地方在呼唤你。你想先去哪？" | 选择引入 |
| 2:00 | 玩家选择域 → 域内探索（首次遭遇 + 首次选择） | 核心玩法 |
| 5:00 | 域内选择完成 → 即时后果 → 可能触发首次回声 | 选择有重量 |
| 7:00 | 收敛点：Navigator "故障" | 悬念建立 |
| 8:00 | Navigator 恢复 + "你来过这里吗？" | 钩子 |
| 8:30 | "……抱歉。不要在意。" | 钩子加深 |
| 9:00 | 第一次会话结束画面：T空间全景，六域闪烁，一句："下一次，你可以去别的地方看看。" | 驱动 Act 2 |

**总时长**：`[PLACEHOLDER: 8-12 分钟]`，符合可行性文档"10-15分钟垂直切片"的要求。

## 九、MVP 垂直切片规格

### 9.1 垂直切片定义

基于可行性文档"下一步建议"第5步——用 Ink/Yarn 格式写一个 10-15 分钟的垂直切片，包含 1 个端口深度 + 1 次跨端口选择。

| 维度 | 规格 |
|------|------|
| **时长** | 10-15 分钟 |
| **包含端口** | 2 个（1 个深度 + 1 个跨端口选择） |
| **推荐端口** | 职业域（深度）+ 诗域（跨端口） |
| **选择理由** | 职业域直接体现"钱多事少离家近"张力；诗域直接体现"生存vs意义"主题；二者交互最能展示系统设计 |
| **格式** | Ink（Inkle）或 Yarn（Yarn Spinner）叙事脚本格式 |
| **包含系统** | 意图分发 + 价值印记 + Navigator 记忆 + 1 次跨域回声 + 收敛点 |

### 9.2 垂直切片内容结构

```
垂直切片流程（职业域深度 + 诗域跨端口）

1. 开场（1min）
   - Navigator 介绍 + 玩家输入意图

2. 域选择（0.5min）
   - 引导玩家选择职业域（但不强制）

3. 职业域深度探索（5min）
   - 第一拍：一份JD——"高薪，但需要长期出差"
   - 第二拍：选择——接受出差？还是拒绝？
     → 接受：生存轴+3，归属轴-1（离家远的代价）
     → 拒绝：自由轴+2，但Navigator说"你放弃了——嗯——很高的分数"
   - 第三拍：即时后果
     → 接受：Navigator 显示一个场景——你的母亲在电话里说"你什么时候回来？"
     → 拒绝：Navigator 显示同一份JD被别人抢走——"它不在了。你确定吗？"

4. 跨端口触发（0.5min）
   - 无论选什么，诗域信号亮起
   - Navigator: "诗域在——嗯——它好像在回应你的选择。要去看看吗？"

5. 诗域跨端口（3min）
   - 第一拍：墙上有一首诗，最后一句被擦掉了
   - 诗的前三句因你在职业域的选择而异：
     → 接受出差：诗关于离别和远方
     → 拒绝出差：诗关于放弃和留下的重量
   - 第二拍：补完这首诗（玩家输入或选择）
   - 第三拍：诗完成后，职业域的JD上出现了诗的最后一句——
     跨域回声触发！你的诗影响了你的工作

6. 收敛点（1min）
   - Navigator "故障"："你……你来过这里吗？"
   - Act 1 钩子
```

### 9.3 垂直切片验收标准

| # | 验收标准 | 检验支柱 |
|---|---------|---------|
| 1 | 玩家在 10-15 分钟内完成完整体验 | — |
| 2 | 玩家做出了至少 2 个有重量的选择 | P1 |
| 3 | Navigator 的 confidence 变化被玩家感知到 | P2 |
| 4 | 玩家在职业域的选择影响了诗域的内容 | P5 |
| 5 | 诗域的诗出现在职业域的JD上（跨域回声） | P1 |
| 6 | 玩家到达收敛点时，对 Navigator 的话有自己的理解 | P5 |
| 7 | 玩家在结束时想"再来一次"或"如果选不同的会怎样" | — |
| 8 | 没有任何一段对话是 AI 生成的（全部手工编写） | P4 |

### 9.4 垂直切片所需资产清单

| 资产类型 | 数量 | 说明 |
|----------|------|------|
| Navigator 对话行 | ~60 句 | 覆盖正常/不确定/诗域/收敛状态 |
| 职业域叙事内容 | 1 个 JD + 2 个选择分支 + 2 个后果场景 | 接受/拒绝出差 |
| 诗域叙事内容 | 2 首半诗（因职业域选择而异）+ 补完选项 | 2 个版本 |
| 跨域回声事件 | 1 个 | 诗出现在JD上 |
| 收敛点对话 | 5-8 句 | Navigator 故障 + 钩子 |
| UI/交互元素 | 意图输入框 + 域选择界面 + confidence 显示 + 诗补完界面 | 最小可玩 |
| 音效/视觉 | Navigator 指示灯动画 + 域过渡效果 + 诗域氛围 | 概念级即可 |

### 10.2 避坑检查清单

在每次设计评审时，逐项检查：

- [ ] **Cyberpunk 检查**：当前设计的分支节点图是否已完整画出？是否有无法收束的死角？
- [ ] **Mass Effect 3 检查**：收敛点是否有质感差异？不同路径的玩家是否会对同一节点有不同理解？

- [ ] **Forspoken 检查**：Navigator 的对话是否在用幽默逃避深度情感？"钱多事少离家近"是否被当作笑料而非生存焦虑？
- [ ] **Fallout 76 检查**：环境叙事是否有活的角色给它语境？Navigator 的对话量是否足够？
- [ ] **Starfield 检查**：当前内容中哪些是手工编写的？哪些是 AI 生成的？核心路径是否 100% 手工？
- [ ] **Anthem 检查**：当前设计是否服务于主题北极星？有没有功能偏离了"生存vs意义"？
- [ ] **Dishonored 2 检查**：后果是否可追溯？玩家能否感知因果链？后果是"惩罚"还是"选择的重量"？
