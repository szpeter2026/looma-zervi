## 一、核心假设与讨论前提

在进入具体数字前，先声明本次预算的**基本假设**。这些假设需要在会议上逐一确认或修正。

### 1.1 时间线假设

| 阶段 | 时间 | 持续 | 核心产出 |
|------|------|------|----------|
| Phase 0 游戏化 MVP | 2026.07.06 – 2026.07.31 | **4 周** | 10 分钟叙事体验可交互原型 + 种子用户验证 |
| 过渡期（决策 + 招募） | 2026.08.03 – 2026.08.14 | **2 周** | Phase 0 数据分析 + 决策门 + Phase 1 团队补齐 |
| Phase 1 功能 MVP + 叙事层 | 2026.08.17 – 2026.10.09 | **8 周** | 小程序上线（叙事嵌入的 AI 求职功能） |
| **合计** | 2026.07.06 – 2026.10.09 | **14 周** | |

> ⚠️ **讨论点 1**: 时间线是否可行？Phase 1 在 10 月中旬上线能否赶秋招尾巴？

### 1.2 人力成本假设

| 假设项 | 采用值 | 说明 |
|--------|--------|------|
| 月工作日 | 21.75 天 | 法定标准 |
| 全职月薪计算 | 月薪 ÷ 21.75 = 日薪 | — |
| 外包/自由职业者日费率 | 市场报价 | 见 §8 薪酬对标 |
| 社保公积金企业部分 | 月薪 × 35%（约数） | 含五险一金企业部分，各地有差异 |
| 全员全职用工起始 | Phase 1 启动后 | Phase 0 期间混合用工（见 §3.2） |
| 办公/设备/杂项 | 另计，不纳入人员预算 | 见 §6 其他费用 |

### 1.3 角色定义

| 角色代号 | 全称 | 职责范围 |
|----------|------|----------|
| **ND** | 叙事设计师 (Narrative Designer) | Navigator 对话编写、六域叙事内容、角色声音支柱维护、Ink/Yarn 脚本 |
| **FE** | 前端工程师 (Frontend Engineer) | 小程序/H5 前端开发、叙事引擎集成（Ink/Yarn runtime）、UI 组件 |

| **QA** | 测试工程师 (QA Engineer) | 功能测试、叙事一致性测试、跨域回声验证、种子用户 playtest 主持 |

### 7.2 外包 vs 内部策略

| 角色 | 建议策略 | 理由 |
|------|----------|------|
| **ND 叙事设计师** | Phase 0: 自由职业者；Phase 1: 内部转岗优先 | 叙事是核心护城河，外部 ND 难以在短时间理解产品哲学；但如果内部无人可转，可考虑与自由职业 ND 签 6 个月合同 |
| **FE2 前端** | 正式招聘 | 小程序开发需要长期维护，外包质量不可控 |
| **BE2 后端** | 正式招聘 | AI 管线是长期基础设施 |
| **PM** | 内部转岗优先 | 需要深度理解产品和叙事设计 |
| **UX / VI** | 外包 | Phase 0-1 设计密度低，长期外包性价比更高 |
| **QA** | Phase 1: 外包 | Phase 1 结束后测试密度下降，无需全职 |

### 7.3 外包风险与缓解

| 风险 | 缓解 |
|------|------|
| 外包 ND 不理解 Navigator 角色声音支柱 | 在合同中要求：前 3 天为"声音支柱对齐期"，交付 5 句样本对话，审核通过后继续 |
| 外包 UX 设计与叙事调性不符 | 提供 GDD §6 Navigator 角色声音支柱 + §5 节点图作为设计 Brief |
| 外包 QA 无法验证叙事一致性 | ND 提供"叙事一致性检查清单"，QA 按清单逐项验证 |

> ⚠️ **讨论点 4**: 是否已有合作过的自由职业 ND？如果没有，招聘周期需要预留多少？

## 九、风险储备金与弹性预算

### 9.1 储备金分级

| 风险等级 | 储备金比例 | 适用阶段 | 触发条件 |
|----------|:---:|------|------|
| **低风险** | 15% | Phase 0、过渡期 | 范围可控、团队精简、产出清晰 |
| **中风险** | 20% | Phase 1 | 多人协作、外包依赖、小程序合规风险 |
| **高风险** | 30% | Phase 1 延期场景 | 如 Phase 0 验收未通过需迭代重测 |

### 9.2 主要风险场景的预算影响

| 风险场景 | 可能性 | 额外成本（元） | 应对 |
|----------|:---:|:---:|------|
| Phase 0 验收不通过，需迭代 2 周重测 | 中 | +15,000 | 从 Phase 0 储备金支出；延期但不进入 Phase 1 |
| ND 中途更换 | 低 | +20,000 | 新 ND 需要 1 周"声音支柱对齐期" |
| 小程序审核被拒 | 中 | +5,000 | 修改 + 重新提交；延期 1-2 周 |
| Phase 1 范围蔓延（新增需求） | 高 | +30,000 – 50,000 | 需严格 scope 管控；超出部分需单独审批 |
| LLM API 成本超预期 | 低 | +5,000/月 | 设置 API 调用预算上限 + 降级方案（规则引擎） |
| 关键招聘延迟 | 中 | 0（时间成本） | 启动招聘提前至过渡期；备选外包名单 |

### 9.3 建议的弹性预算范围

| 预算方案 | 金额（元） | 说明 |
|----------|:---:|------|
| **底线预算**（无储备金、无招聘延迟） | 431,550 | 理想状态，所有假设成立 |
| **基准预算**（含 15-20% 储备金） | **515,198** | **本次提案采用的方案** |
| **上限预算**（含 30% 储备金 + 1 个风险场景触发） | 590,000 | Phase 0 迭代 + 1 个关键延迟同时发生 |

> ⚠️ **讨论点 6**: 公司对预算弹性空间的态度？是按基准预算审批，还是按上限预算预留？
